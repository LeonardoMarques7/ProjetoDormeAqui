# Regras de UI/UX

Antes de alterar qualquer tela, siga estas regras obrigatórias.

## Hierarquia visual

1. A informação mais importante deve aparecer primeiro.
2. O usuário deve entender a tela em até 5 segundos.
3. Cada seção deve ter um objetivo claro.
4. Evite mostrar tudo com o mesmo peso visual.
5. KPIs devem responder perguntas práticas, não apenas exibir números.

## Layout

- Usar grid consistente.
- Evitar componentes desalinhados.
- Evitar excesso de scroll em informações prioritárias.
- Agrupar informações relacionadas.
- Usar espaçamento generoso entre seções.
- Usar cards somente quando eles ajudarem a separar contexto.
- Evitar colocar muitos cards pequenos lado a lado sem prioridade.

## Cards

- Cards devem ter título claro.
- Cards devem ter uma informação principal.
- Cards podem ter uma informação secundária, mas nunca competir com a principal.
- Evitar cards com muito texto.
- Usar ações visíveis quando o card exigir decisão.

## Tipografia

- Títulos de página devem ser fortes e objetivos.
- Subtítulos devem explicar o valor da tela.
- Números importantes devem ter destaque visual.
- Textos auxiliares devem ser menores e neutros.
- Evitar tudo em negrito.

## Cores

- Usar base neutra.
- Usar cor primária somente para CTA, destaque e estados importantes.
- Usar verde apenas para sucesso/positivo.
- Usar vermelho apenas para alerta/problema.
- Usar amarelo/laranja apenas para atenção.
- Não usar várias cores decorativas sem função.

## Estados

Toda tela importante deve prever:

- loading
- vazio
- erro
- dados reais
- dados insuficientes

## Responsividade

- Mobile não deve ser apenas desktop espremido.
- Cards devem virar blocos verticais.
- Tabelas devem virar cards ou listas compactas.
- A navegação lateral pode virar drawer/menu.
